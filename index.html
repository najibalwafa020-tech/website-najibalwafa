Apa itu GitHub Pages?

GitHub Pages adalah layanan hosting website statis yang gratis dari GitHub. Anda dapat menhost website personal, project documentation, atau blog dengan mudah langsung dari repository GitHub Anda.

💰 Gratis

Hosting tanpa biaya bulanan selamanya

⚡ Cepat

CDN global untuk performa optimal

🔒 Aman

HTTPS otomatis untuk semua situs

📱 Responsif

Bekerja sempurna di semua perangkat

🔧 Panduan Setup

Langkah 1: Buat Repository Baru

Di GitHub:

1. Klik tombol "+" di sudut atas kanan dan pilih "New repository"

2. Beri nama repository: <username>.github.io

Contoh: najibalwafa.github.io

3. Pastikan repository bersifat "Public"

4. Klik "Create repository"

Langkah 2: Clone Repository

git clone https://github.com/username/username.github.io.git cd username.github.io

Langkah 3: Tambah File HTML

Buat file index.html:

Di folder repository lokal, buat file bernama index.html dengan konten HTML Anda.

Langkah 4: Push ke GitHub

git add . git commit -m "Initial commit" git push origin main

Langkah 5: Akses Website

Website Anda siap diakses di:

https://username.github.io

Tunggu beberapa menit untuk deployment selesai.

Pengaturan Lanjutan

Gunakan Domain Custom:

1. Pergi ke Settings repository → Pages

2. Di bagian "Custom domain", masukkan domain Anda

3. Konfigurasi DNS di provider domain Anda

💡 Tips: Anda dapat menggunakan Jekyll untuk membuat blog atau website yang lebih dinamis. GitHub Pages mendukung Jekyll secara native.

📊 Bandwidth & Limitasi

Quota GitHub Pages

Aspek Batas Keterangan
Ukuran Repository Unlimited Simpan file sebanyak yang Anda butuhkan
Bandwidth/Bulan 100 GB* Sangat cukup untuk sebagian besar website
Build per Jam 10 builds Untuk Jekyll builds
Ukuran File 100 MB max Per file individual
Uptime 99.9% SLA dijamin GitHub

Estimasi Penggunaan Bandwidth

Contoh perhitungan untuk website kecil:

• Ukuran rata-rata halaman: 500 KB

• Pengunjung per hari: 1000

• Penggunaan harian: 500 MB

• Penggunaan per bulan: 15 GB

Kesimpulan: Masih jauh di bawah limit 100 GB/bulan ✓

Tips Mengoptimalkan Bandwidth

📦 Kompresi Gambar

Gunakan tools seperti TinyPNG untuk mengurangi ukuran gambar

⚡ Minifikasi CSS/JS

Kurangi ukuran file dengan minification

🗜️ Gzip Compression

GitHub Pages otomatis menggunakan gzip

📱 Lazy Loading

Muat gambar hanya saat diperlukan

ℹ️ Catatan: Bandwidth 100 GB/bulan sangat generous. Untuk context: 1 juta page views dengan average 100 KB per halaman = 100 GB. Website kebanyakan tidak akan mencapai limit ini.

🔌 Backend & Dinamis Content

Batasan GitHub Pages

⚠️ Penting: GitHub Pages hanya mendukung website statis (HTML, CSS, JavaScript client-side). Tidak ada backend server yang berjalan.

Solusi untuk Backend

1. Menggunakan API Eksternal

Cara Kerja:

• Website statis yang di-host di GitHub Pages

• Backend API di server lain (Firebase, Heroku, dll)

• Berkomunikasi via AJAX/Fetch API

Contoh:

// fetch('/api/data') dari backend lain fetch('https://api.example.com/data') .then(res => res.json()) .then(data => console.log(data));

2. Serverless Functions

Platform Fitur Harga Gratis
Netlify Functions AWS Lambda backend 125k invocations/bulan
Vercel Serverless Edge & serverless functions Unlimited eksekusi
Firebase Functions Google Cloud backend 2M invocations/bulan
AWS Lambda Function computing 1M invocations/bulan

3. Database Solusi

🔥 Firebase Realtime DB

Database real-time dari Google. Gratis untuk penggunaan kecil.

☁️ Firebase Firestore

Database NoSQL modern dengan query powerful. 50k reads gratis/hari.

🛢️ Supabase

PostgreSQL backend as a service. Gratis dengan banyak fitur.

⚙️ MongoDB Atlas

Cloud database. Tier gratis dengan 512MB storage.

Arsitektur Rekomendasi

📐 Stack Umum:

• Frontend: GitHub Pages (HTML/CSS/JS)

• Backend: Firebase, Serverless Functions, atau REST API

• Database: Firebase Firestore, MongoDB, PostgreSQL

• CDN: GitHub Pages built-in atau Cloudflare

Contoh: Aplikasi Todo dengan Backend

<button onclick="addTodo()">Add Todo</button> <script> function addTodo() { fetch('https://your-api.com/todos', { method: 'POST', body: JSON.stringify({ title: 'New Todo' }) }) .then(res => res.json()) .then(data => console.log('Todo added:', data)); } </script>

Alasan Memilih Backend Eksternal

  • ✓ GitHub Pages gratis tetap gratis (hanya frontend)
  • ✓ Backend scalable dengan mudah
  • ✓ Database terpisah dan aman
  • ✓ Dapat menambah fitur dinamis kapan saja
  • ✓ Banyak pilihan free tier untuk memulai
💡 Rekomendasi: Untuk project kecil, gunakan Firebase. Gratis, mudah setup, dan scalable. Untuk project besar, pertimbangkan Vercel atau Netlify dengan backend custom.
